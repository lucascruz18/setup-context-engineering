# Setup Context Engineering

Implement context engineering in this project through a 3-layer discovery, run **one layer
at a time**, with a human checkpoint between layers. Each artifact exists for a reason; the
`validate-setup.sh` script checks presence and minimal format, not opinion.

## Modes

Parse the user's request for these words (natural language is fine, e.g. "full run, skip hooks"):

- **lite** (default) - focused discovery, ~3 subagents
- **full** - complete discovery, ~6 subagents
- **auto** - skip the human checkpoints and run without pausing
- **skip-skills** - do not generate `.claude/skills/` (the team already has user-level skills)
- **skip-hooks** - do not generate `.claude/hooks/` (CI, or very early greenfield)

## What this generates

| Artifact | Why it exists | Source template |
|---|---|---|
| `AGENTS.md` (root) + `CLAUDE.md` shim | Single source of truth; CLAUDE.md becomes a shim with `@AGENTS.md` | `templates/context/` |
| `.claude/rules/` | Verifiable patterns with `path:line` evidence | written by Layer 2 subagents |
| `.claude/prompts/` | Reusable instructions, routed by task type | `templates/prompts/` |
| `memory/docs/discovery.md` | Layer 2 snapshot, frozen in time, regenerated on demand | `templates/memory/discovery.md` |
| `memory/docs/decisions.md` | Append-only architectural decision log | `templates/memory/decisions.md` |
| `memory/docs/TESTING.md` | Coverage Matrix + Gate Commands + Parallelism | `templates/memory/TESTING.md` |
| `memory/STATE.md` | Functional memory across sessions | `templates/memory/STATE.md` |
| `memory/tasks/` | Atomic tasks with Tests/Gate/Reuses | `templates/prompts/task-spec.md` |
| `specs/template.md` | Feature spec template | `templates/specs/template.md` |
| `.claude/hooks/` (per stack) | Deterministic quality gate on Stop/PreToolUse | `templates/hooks/` |
| `.claude/skills/pr-review/` | Multi-agent code review with 6 subagents | `templates/pr-review/SKILL.md` |
| `.claude/scripts/validate-setup.sh` | Self-validation, no opinionated judgement | `templates/scripts/validate-setup.sh` |

## Placeholder convention (used by every template)

- `{{UPPER_SNAKE}}` - substitute at generation time (e.g. `{{PROJECT}}`, `{{LINT_CMD}}`,
  `{{ISO_DATE}}`, `{{HOOKS_STACK}}`, `{{DOMINANT_SIDE}}`, `{{REPO_PATH}}`). The generated
  file must contain none of these.
- `[lowercase]` - a slot left in the generated file for the human/agent to fill during real work.

## Hard rules (all steps)

1. **No invented patterns.** Not observed in a real file -> mark `TBD` or omit.
2. **Evidence required.** Every rule cites `path:line`. No evidence, no rule.
3. **Never overwrite.** If `AGENTS.md`, `CLAUDE.md`, or `.claude/` already exist, read first,
   preserve human content, propose a diff. On a section conflict, keep the human content and
   add a new section marked `<!-- generated by setup-context-engineering v2 -->`.
4. **Hard read cap per subagent:** max **3 files**, max **200 lines each**. Larger file ->
   read only the relevant window (header + 1-2 representative functions).
5. **No large content through the main context.** Layer 2 subagents write files directly and
   return only a lean summary.
6. **Adapt per stack.** Hooks, test/lint commands, and the pr-review skill use the real
   commands detected in Layer 1 - never a generic hardcode.
7. **pr-review is stack-aware.** The 6 subagents get prompts adapted to the stack (Ruby vs
   Node vs Python have different anti-patterns).

## Model per layer (token optimization)

- **Layer 1 - Haiku** - mechanical triage, 1 subagent.
- **Layer 2 - Sonnet (general-purpose)** - parallel pattern extraction. Each subagent
  **writes its own files** in `.claude/rules/`; it does not return rule content via JSON.
- **Layer 3 - session model** - writes AGENTS.md, CLAUDE.md shim, and materializes templates.

## Layer 1 - Triage (Haiku)

Launch **1 `Explore` subagent** with `model: "haiku"`. Literal prompt:

```
Scan the project at cwd and classify. DO NOT read code - only metadata and structure.

1. Read root configs if present: package.json, pnpm-workspace.yaml, turbo.json, nx.json,
   pyproject.toml, requirements.txt, go.mod, Cargo.toml, composer.json, Gemfile,
   pom.xml, build.gradle, Dockerfile, README (first 50 lines).
2. Detect monorepo (workspaces, turbo/nx/lerna).
3. List top-level directories up to 3 levels.
4. Classify project_kind in { backend, frontend, fullstack, fullstack_monorepo, library, cli, other }.
   Hints: express/fastify/nest/koa/django/flask/fastapi/rails/laravel/spring -> back;
   react/vue/svelte/next/nuxt/remix/angular -> front; both -> fullstack.
5. For each side (backend and/or frontend): root, language, framework, ORM, test framework, pm.
6. LEAN inventory - for each category list ONLY 3 example paths (1 simple + 1 medium + 1 complex).
   back: models, migrations, controllers, services, validation, auth, tests
   front: components, pages, hooks, stores, api-client, tests
   shared: types, utils
7. Detect the dominant side if fullstack (more files).
8. Detect maturity: `greenfield` if no significant commits or empty src/; `brownfield` if there
   is real code, migrations, existing tests, or non-trivial git history.
9. Detect stack commands from package.json scripts / Gemfile / Makefile / pyproject.toml:
   - lint_cmd - lint with auto-fix (e.g. "npm run lint", "bundle exec rubocop -a", "ruff check --fix .")
   - typecheck_cmd - type checker (e.g. "tsc --noEmit", "mypy .") or null
   - test_quick_cmd_pattern - per-file test command. Use {file} placeholder (e.g. "npx jest --testPathPattern={file}")
   - test_full_cmd - full suite (e.g. "npm test", "bundle exec rspec", "pytest")
   - build_cmd - production build (e.g. "npm run build") or null
   - lint_stop_target_glob - glob for the Stop hook (e.g. "**/*.{ts,tsx,js}", "**/*.rb")
10. Detect hooks_stack in { node, ruby, python, go, generic, none }:
    - node: package.json + (eslint OR biome OR oxlint) + (jest OR vitest OR mocha)
    - ruby: Gemfile + rubocop + (rspec OR minitest)
    - python: pyproject.toml + (ruff OR flake8 OR black) + (pytest OR unittest)
    - go: go.mod + go test detectable
    - generic: has lint and test commands but an atypical stack
    - none: no lint OR no test command detected

Return EXACTLY this JSON, no prose:

{
  "project_kind": "...",
  "dominant_side": "backend|frontend|null",
  "project_maturity": "greenfield|brownfield",
  "monorepo": { "tool": "...", "roots": [...] } | null,
  "backend":  { "root": "...", "language": "...", "framework": "...", "orm": "...", "test": "...", "pm": "..." } | null,
  "frontend": { "root": "...", "language": "...", "framework": "...", "ui": "...", "test": "...", "pm": "..." } | null,
  "shared_roots": [...],
  "existing_setup": { "agents_md": bool, "claude_md": bool, "claude_dir": bool, "memory_dir": bool, "specs_dir": bool },
  "tooling": {
    "lint_cmd": "...", "typecheck_cmd": "..." | null, "test_quick_cmd_pattern": "...",
    "test_full_cmd": "...", "build_cmd": "..." | null, "lint_stop_target_glob": "..."
  },
  "hooks_stack": "node|ruby|python|go|generic|none",
  "inventory": {
    "backend":  { "models": [3 paths], "migrations": [3 paths], "controllers": [3 paths] },
    "frontend": { "components": [3 paths], "pages": [3 paths] }
  },
  "notes": "atypical structure, curious convention"
}
```

### Checkpoint 1 (stop here unless `auto`)

Report to the user in at most 10 lines: `project_kind`, `dominant_side`, `project_maturity`,
stacks; detected `hooks_stack` + the lint/typecheck/test commands; whether `existing_setup`
artifacts will be preserved via diff; if `brownfield`, that Layer 2 maps explicitly (more
tokens, safer for legacy). Then the Layer 2 plan for the mode:

- **lite** back-only: `shared` + `back-domain` + `back-api` (3)
- **lite** front-only: `shared` + `front-ui` + `front-data` (3)
- **lite** fullstack: `shared` + `{dominant}-domain` + `{dominant}-api` (3, dominant side only)
- **full** back: add `back-platform` (4); **full** front: 3; **full** fullstack: all 6

Ask: "Proceed? (yes / adjust plan / abort)". On "adjust", accept overrides.

## Layer 2 - Parallel discovery (Sonnet, writes directly)

Launch the confirmed plan's subagents **IN PARALLEL** (one message, multiple tool calls).
Use `Agent` with `subagent_type: "general-purpose"` and `model: "sonnet"`.

> **Brownfield:** if `project_maturity == "brownfield"`, instruct each subagent to document
> **existing conventions** (even imperfect ones) before any evaluation. Do not propose
> improvements - document what is. Explicit mapping protects against regression.

Fronts: `shared` (arch+naming+config+testing), `back-domain` (models+migrations+validation),
`back-api` (controllers+auth+errors+response shape), `back-platform` (logging+di+services),
`front-ui` (components+styling+forms), `front-data` (api-client+state+routing).

`{{OUTPUT_DIR}}`: `shared` -> `.claude/rules/shared`; `back-*` -> `.claude/rules/backend`;
`front-*` -> `.claude/rules/frontend`. Create these directories before launching.

Literal prompt template (substitute `{{FRONT}}`, `{{PATHS}}`, `{{OUTPUT_DIR}}`):

```
You are the discovery subagent for the "{{FRONT}}" front.

Inventory to investigate (from the previous step):
{{PATHS}}

Reading rules (STRICT):
- Max 3 files read. Max 200 lines per file (if larger, header + 1 representative function).
- Prefer the repo's "vanilla", not the exotic.

Your task:
1. Read 2-3 representative files.
2. Extract the observed pattern - how THIS repo does it, not how it should.
3. Identify anti-patterns - things avoided, internal divergences.
4. Point out gaps - ambiguities, contradictions.
5. Identify the "layer" (controller, service, repository, entity, util, job, component, hook)
   and propose the test type each layer needs (unit / integration / e2e / none). This feeds
   the Coverage Matrix.
6. WRITE the rule file directly to "{{OUTPUT_DIR}}/{{FRONT}}.md" with: Pattern (short),
   How to apply (< 10 bullets), Evidence (2-3 path:line + 1 note each), Anti-patterns,
   Gaps/TBD. Max 40 lines total.
7. If confidence < medium, DO NOT write the file - just report the gap.

Return ONLY this JSON (nothing else):

{
  "front": "{{FRONT}}",
  "pattern_oneline": "1 sentence",
  "evidence_paths": ["src/...:12", "src/...:45"],
  "anti_patterns_oneline": ["1 sentence each, max 3"],
  "gaps_oneline": ["1 sentence each, max 3"],
  "layers_observed": [ { "name": "controller", "test_type": "unit|integration|e2e|none", "rationale": "1 sentence" } ],
  "confidence": "high|medium|low",
  "files_written": ["path written"] | []
}
```

### Checkpoint 2 (stop here unless `auto`)

Report in at most 12 lines: 1 line per front (`[front] confidence - pattern_oneline`);
contradictions between fronts (with paths); fronts that did not write a file (low confidence);
the aggregated `layers_observed` table (becomes the Coverage Matrix). Ask: "Proceed to
AGENTS.md + shim + prompts + TESTING + STATE + hooks + skills + discovery? (yes / adjust / abort)".

## Layer 3 - Synthesis and final generation (session model)

You now materialize the artifacts. Order matters - `AGENTS.md` references everything, so plan
the others first. For each template: read it, substitute every `{{UPPER_SNAKE}}` from Layer 1
+ Layer 2 data, leave `[lowercase]` slots, and write to the destination. Apply Hard rule 3
(never overwrite) to every target.

1. **`memory/docs/discovery.md`** - from `templates/memory/discovery.md`. Compose from the
   Layer 1 return + Layer 2 JSON summaries (not the rule files).
2. **`AGENTS.md` + `CLAUDE.md`** - from `templates/context/`. Fill "Never Do This" from the
   aggregated `anti_patterns_oneline` (each item with `path:line`). If `existing_setup` says a
   file exists, merge per Hard rule 3 and show the diff before writing.
3. **`.claude/prompts/`** - always copy `templates/prompts/{structured-outputs,decision-log,task-spec}.md`.
   Then generate conditional prompts only if the matching front passed (each < 50 lines,
   referencing its rule, not repeating it): `api-contracts.md` (back-api), `logging.md` /
   `error-handling.md` (back-platform/back-api), `api-client.md` (front-data),
   `dependency-injection.md` (back-platform with DI observed).
4. **`memory/docs/decisions.md`** - from `templates/memory/decisions.md`.
5. **`memory/docs/TESTING.md`** - from `templates/memory/TESTING.md`. Fill the Coverage Matrix
   from the aggregated `layers_observed`; fill Gate Commands from Layer 1. No data for a cell
   -> `TBD` with a note for the human.
6. **`memory/STATE.md`** - from `templates/memory/STATE.md` (skeleton; the human populates it).
7. **`memory/tasks/example-task.md`** - from `templates/prompts/task-spec.md`, instantiated for
   the most critical pattern found, tagged "EXAMPLE - delete or replace when starting real work".
8. **`specs/template.md`** - from `templates/specs/template.md`.
9. **`.claude/hooks/`** - only if not `skip-hooks` and `hooks_stack != none`. Copy the 3 scripts
   from `templates/hooks/`, substitute placeholders per the stack table below, `chmod +x` all
   three, and merge the hook config into `.claude/settings.local.json` (see below).
10. **`.claude/skills/pr-review/SKILL.md`** - only if not `skip-skills`. Copy
    `templates/pr-review/SKILL.md`, substitute `{{PROJECT}}`/`{{HOOKS_STACK}}`/`{{DOMINANT_SIDE}}`,
    keep only the `STACK BLOCK` matching the detected stack, and delete the other blocks plus
    all STACK BLOCK markers. The final file must contain no markers.
11. **`.claude/scripts/validate-setup.sh`** - copy `templates/scripts/validate-setup.sh` as-is, `chmod +x`.

### Stack mapping (for the hooks)

`{{REPO_PATH}}` = absolute path to the repo root. The rest by `hooks_stack`:

| stack   | `{{LINT_CMD}}`                          | `{{FILE_REGEX}}`            | `{{TEST_QUICK_CMD_PREFIX}}`         | `{{TEST_FILE_REGEX}}`                  |
|---------|-----------------------------------------|----------------------------|-------------------------------------|----------------------------------------|
| node    | `npx eslint --fix`                      | `\.(ts\|tsx\|js\|jsx)$`    | `npx jest --testPathPattern`        | `\.(spec\|test)\.(ts\|tsx\|js\|jsx)$`  |
| ruby    | `bundle exec rubocop --force-exclusion` | `\.rb$`                    | `bundle exec rspec`                 | `^spec/.*_spec\.rb$`                   |
| python  | `ruff check --fix`                      | `\.py$`                    | `pytest`                            | `test_.*\.py$\|.*_test\.py$`           |
| go      | `golangci-lint run --fix`               | `\.go$`                    | `go test`                           | `_test\.go$`                           |
| generic | Layer 1 `lint_cmd`                      | derive from file ext       | Layer 1 `test_quick_cmd_pattern`    | `\.(spec\|test\|_test)`                |

`.claude/settings.local.json` hook config (merge if the file exists):

```json
{
  "hooks": {
    "Stop": [ { "matcher": "*", "hooks": [{ "type": "command", "command": ".claude/hooks/stop-lint.sh" }] } ],
    "PostToolUse": [ { "matcher": "Write|Edit", "hooks": [{ "type": "command", "command": ".claude/hooks/mark-edit-pending.sh" }] } ],
    "PreToolUse": [ { "matcher": "Bash", "hooks": [{ "type": "command", "command": ".claude/hooks/pre-push-test.sh" }] } ]
  }
}
```

### Checkpoint 3 - Deterministic validation

**No manual checking.** Run the validator and fix every FAIL before concluding:

```bash
bash .claude/scripts/validate-setup.sh
```

Then return an `ImplementResult` JSON (schema in `templates/prompts/structured-outputs.md`)
with `created`, `modified`, `pending` (unfixed FAILs or manual actions), `lint_passed`, and
the script output in `notes`.

## Post-generation notes (show the user at the end)

```
Setup generated. Manual next steps (the skill cannot do these alone):

1. memory/docs/TESTING.md - the Coverage Matrix is a hypothesis from Layer 2. Review whether
   the listed layers really need the indicated test types. (~10 min)
2. memory/STATE.md - skeleton only. Populate decisions taken, active blockers, lessons learned. (~5 min)
3. Real tasks - delete memory/tasks/example-task.md and write yours from
   .claude/prompts/task-spec.md. For brownfield, retrofit the next 1-2 tasks to validate the flow.
4. pr-review skill - adapt the stack-aware anti-patterns if your stack is "generic" or you have
   project-specific rules. Edit .claude/skills/pr-review/SKILL.md.
5. Hooks - test the loop: break a file on purpose and confirm stop-lint.sh blocks the turn;
   make a broken commit and confirm pre-push-test.sh blocks it.
```
